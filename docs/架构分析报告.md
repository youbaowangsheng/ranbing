## 三、前端结构

### 3.1 Web 端（TemplateView）

- **技术栈**：Django TemplateView + 原生 HTML/JS
- **路由**：通过 `pages/` URL 前缀，所有页面返回 HTML 模板
- **认证**：Session + CSRF（`django.middleware.csrf.CsrfViewMiddleware`）
- **登录页**：`/pages/login/` → `page_login.html`
- **主要页面**：
  - `page_P_home.html` - 首页
  - `page_K_v2_supply_demand.html` - 供需广场
  - `page_S_v2_publish_supply.html` - 发布供需
  - `page_L_activity_list.html` - 活动列表
  - `page_N_v2_community.html` - 社群
  - `page_A_profile_view.html` - Profile 查看
  - `page_B_profile_edit.html` - Profile 编辑
  - `page_Q_ai_assistant.html` - AI 助手

### 3.2 微信小程序端

- **技术栈**：原生微信小程序（无框架）
- **TabBar 页面**（5个）：
  1. 首页 `/pages/home/home`
  2. 供需 `/pages/supply-demand/supply-demand`
  3. 活动 `/pages/activity/activity`
  4. 社群 `/pages/community/community`
  5. 我的 `/pages/profile/profile`

- **其他页面**（17个）：
  - `login/login`, `login/register` - 登录注册
  - `publish/publish` - 发布供需
  - `my-posts/my-posts` - 我的发布
  - `ai-assistant/ai-assistant` - AI 助手
  - `supply-detail/supply-detail` - 供需详情
  - `activity-detail/activity-detail` - 活动详情
  - `community-detail/community-detail` - 社群详情
  - `messages/messages`, `chat/chat` - 消息聊天
  - `search/search` - 搜索
  - `profile-edit/profile-edit`, `profile-view/profile-view` - Profile 编辑/查看
  - `contacts/contacts`, `contact-tags/contact-tags` - 联系人
  - `cards/cards`, `card-edit/card-edit` - 名片

- **API 服务层**（`services/api.js`）：
  - `API_BASE = 'https://www.asiamlhk.com/api/v1'`
  - JWT Bearer Token 认证（从 `wx.getStorageSync('token')` 获取）
  - 401 响应 → 清除 token → 跳转登录页

```
User (1:1) ──── Profile

   ┌───────────────┼───────────────┐
    │               │               │
```


## 六、认证机制

### 6.1 双轨认证

| 前端 | 认证方式 | 路由 |
|------|---------|------|
| **小程序** | JWT Bearer Token | `/api/v1/*` |
| **Web** | Session + CSRF | `/pages/*` |

### 6.2 JWT 认证流程

```
1. 用户登录 → POST /api/v1/auth/login (phone + code/password)
2. 后端验证 → 生成 JWT access_token + refresh_token
3. 小程序存储 token → wx.setStorageSync('token', token)
4. 后续请求 Header → Authorization: Bearer <token>
5. JWTAuthentication 中间件验证 token
```

### 6.3 登录方式

1. **短信验证码登录**：`/api/v1/auth/login`（phone + code）
2. **微信登录**：`/api/v1/auth/wx_login`（code → openid → user）
3. **邮箱密码登录**：`/api/v1/auth/email_login`（运营控制台用）
4. **Session登录**：`/pages/login/`（Web 端）

### 6.4 验证码机制

- 存储：Redis（`code:{phone}:{type}`，5分钟有效）
- 发送：阿里云 SMS / 腾讯云 SMS / Mock
- 特殊验证码：`000000` 可绕过验证（开发环境）

## 七、部署方式

### 7.1 技术栈

| 组件 | 技术 |
|------|------|
| **后端** | Django 4.x + Django REST Framework |
| **数据库** | PostgreSQL 15+ (需 pgvector 扩展) |
| **缓存/队列** | Redis + Celery |
| **AI** | DeepSeek API (`text-embedding-3-small`, `deepseek-chat`) |
| **短信** | 阿里云/腾讯云 SMS |
| **前端** | 微信小程序 + Django HTML Templates |


### 7.3 数据库 Schema

- PostgreSQL 需要安装 `uuid-ossp` 和 `vector` 扩展

- 触发器：自动更新 `updated_at`、维护 `conn_count` 冗余计数

### 7.4 当前环境

- 开发环境使用 SQLite（`~/ranbing/backend/db.sqlite3`）
- API Base URL：`https://www.asiamlhk.com/api/v1`（小程序）
- Web域名：`https://asiamlhk.com`

## 八、关键设计亮点

1. **双端分离**：Web 端 TemplateView + 小程序 DRF API，共用同一后端
2. **向量搜索**：pgvector 支持基于 embedding 的相似度匹配
3. **AI 增强**：DeepSeek 驱动的标签推荐、供需匹配、质量评分
4. **多维标签**：L1~L5 五层标签体系，支持供给/需求类型区分
5. **活跃度评分**：AI 计算用户活跃度，用于推荐排序
6. **灵活认证**：JWT + Session 双轨，微信登录集成
